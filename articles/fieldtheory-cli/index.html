fieldtheory cli

sync your x bookmarks locally so your ai agent can actually read them. no api. no oauth.
3 min read from @raysterminal

twitter killed their api for most users, which means if you want to export your bookmarks you are stuck with manual scrolling or third-party services that want your password. fieldtheory cli is a command-line tool that logs into x as you, navigates to your bookmarks, and extracts the full list. no api keys. no oauth dance. no monthly subscription. it runs on your machine and keeps your credentials local.

the tool is built on playwright, which means it controls a real browser. it opens x, handles the login flow including two-factor authentication, navigates to the bookmarks tab, and scrolls until it has captured every saved tweet. the output is structured json with the tweet text, author, date, url, and any media attachments. from there you can pipe it into a database, a markdown file, or another tool.

--format json, markdown, or csv. default is json with full metadata.
--since filter by date range. only extract bookmarks after a given date.
--media-only export only tweets that contain images or video attachments.
--watch run on a schedule and append new bookmarks to an existing file. continuously updating.
--dedupe skip bookmarks that already exist in the output file. idempotent syncs.

the use case is straightforward: you have years of bookmarks on x and you want to own that data. maybe you are researching a topic and need to tag your saved threads. maybe you are leaving the platform and want an archive. maybe you run a newsletter and your bookmarks are your raw material. in every case, the official api is not an option, and the manual alternative is not realistic once you pass a few hundred saved items.

the cli has flags for filtering by date range, exporting only media tweets, and deduplicating against a previous run. there is also a watch mode that runs on a schedule and appends new bookmarks to an existing file. this turns your x bookmarks into a continuously updating dataset without any manual intervention after the initial setup.

the project is open source and the core logic is simple enough to audit. it does not send data to any server, it does not inject ads, and it does not modify your account. it only reads the bookmarks page and writes to a file you specify. the credentials live in a local sqlite database that you can delete at any time.

browser automation is fragile by nature. x changes their ui frequently, and any selector-based tool breaks when the dom shifts. fieldtheory handles this by using semantic role-based selectors where possible and falling back to visual anchors when the structure changes. the maintainers track x ui updates and push fixes within days. if you pin the version, you can skip updates until you need them.

the broader point is that platform data should be portable. x, like most social networks, makes extraction difficult because your attention is their product. tools like fieldtheory cli restore the balance. they let you pull your data out in a usable format, on your own hardware, without asking permission.
