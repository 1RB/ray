pascal editor

a full 3d building editor that runs in your browser

someone just open sourced a full 3d building editor that runs in the browser. no autocad, no revit, no five thousand dollar licenses. it is called pascal editor, built with react three fiber and webgpu for near-native speed.

the features are what you would expect from professional architecture software, except it loads in a tab. real-time building design with parametric walls, floors, roofs, and openings. live material editing. first-person walkthrough mode. and it exports to standard formats so you can take your model elsewhere when you are done.

"the gap between 'i have an idea for a building' and 'here is a 3d model' just collapsed to a single browser tab."

the technical stack is worth noting. react three fiber handles the 3d scene graph. webgpu provides the rendering backend, which matters because webgl is now legacy and webgpu gives you compute shaders, meaning you can run geometry operations on the gpu directly. the result is smooth interaction even with complex models, something that was impossible in a browser even two years ago.

the interface follows the blender convention: middle mouse to orbit, scroll to zoom, shift to pan. this is a deliberate choice. anyone who has spent time in 3d software already has the muscle memory, and for new users there is a massive ecosystem of tutorials to draw from. the learning curve is not flat, but it is a known curve.

what makes pascal editor different from the established tools is not just the price. it is the speed of iteration. in revit, changing a wall thickness can trigger a regeneration cascade that takes seconds. in pascal editor, the same operation is instantaneous because the geometry pipeline is built for reactivity from the ground up. when you are sketching, those seconds matter. they are the difference between staying in flow and losing the idea.

the open source release is complete. the repository includes the full source, a live demo, and documentation for extending the editor with custom components. the license is permissive, meaning you can embed it in commercial products or fork it for specialized use cases. there is already discussion in the issues about adding ifc export support, which would make it compatible with the rest of the architecture industry.

the broader implication is that professional-grade 3d software is now a commodity. the moat of the incumbents was never the algorithms, which are well documented in textbooks. it was the distribution, the certification pipelines, and the lock-in of proprietary file formats. a browser-based editor with open formats breaks all three.

for architects, this means you can sketch in pascal editor, present in the same tab, and export to your contractor without buying a license. for developers, it means you can embed building design into any web application. for students, it means you can learn parametric modeling without a lab computer.

the project is early. there are bugs, missing features, and the documentation is thin. but the foundation is solid. the rendering is fast, the geometry is clean, and the architecture is extensible. if you have ever wanted to design a building and been blocked by software costs or operating system requirements, this is the tool that removes those barriers.
