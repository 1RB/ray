warp goes open source

the rust-based terminal is now fully open source under apache 2.0
3 min read from @raysterminal

warp, the terminal built in rust with a modern interface and collaborative features, is now fully open source. the entire codebase is on github under the apache 2.0 license, including the core terminal emulator, the warp drive cloud features, and the ai assistant. this is a significant shift for a company that raised nearly seventy million dollars in venture funding.

the decision to open source was announced alongside warp's public launch. the team argues that a terminal is infrastructure, and infrastructure should be inspectable. users have been asking for this since the first private beta. the concern was never about the ui, which was always polished. it was about trusting a closed-source program with shell access to your machine. that concern is now gone.

core terminal emulator
the full rust source for the pty, shell integration, and gpu-accelerated renderer. metal on macos, vulkan on linux.
warp drive
team collaboration features including shared command palettes, notebooks, and environment variables. the sync protocol and server implementation are both published.
warp ai
the assistant that translates natural language to shell commands, explains flags, and suggests fixes. integrated into the input line with context from the terminal session.

the technical architecture is built on a custom gpu renderer. the terminal surface is drawn with metal on macos and vulkan on linux, which means scrolling is smooth, ligatures render correctly, and the interface stays responsive even when streaming thousands of lines of output. the rust core handles the pty, the shell integration, and the protocol that syncs state between local and remote sessions.

the collaborative features are what differentiate warp from other modern terminals. warp drive lets teams share command palettes, notebooks, and environment variables. when you join a team, you get access to the shared command history, which means you can find the exact incantation your teammate used to deploy last tuesday without searching slack. the notebooks are particularly useful: executable markdown documents that mix shell commands, output, and prose. they function as living runbooks that anyone on the team can execute.

open sourcing the full codebase, including the cloud components, is unusual. most companies open source the client and keep the server proprietary. warp took the opposite approach, publishing the warp drive sync protocol and the server implementation. this means you can self-host the collaborative features if you do not want to use their cloud. the team is betting that the managed service will still win on convenience, and that transparency will attract enterprise customers who need to audit their tools.

the build process is straightforward for a rust project. clone the repo, install the dependencies, and compile. the repository is large because it includes the ui assets, the font files, and the machine learning models for the ai features. but the core terminal compiles quickly, and the team provides clear documentation for getting a development build running.

there are tradeoffs. warp is still macos and linux only, with windows support planned. the interface is opinionated, with a block-based output system that some users love and others find intrusive. and while the code is open, the roadmap and governance are controlled by the company, not a community foundation. this is open source in the pragmatic sense, not the ideological sense.

for users who have been waiting to switch from iterm or alacritty, the open source release removes the final barrier. you can audit the code, contribute fixes, and run a fork if you disagree with the direction. for the team, it is a bet that transparency will build trust faster than features alone. in a category as personal as the terminal, trust matters.
